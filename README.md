## Getting Started

### Prerequisites

- **Node.js:** `v24.0.0`
- **npm:** `v13.0.0`

#### 1. Clone the Repository

```bash
git clone https://github.com/Clarioscope-AI/bralvio.io.git
```
#### 2. Navigate to the Project

```bash
cd bralvio.io
```
#### 3. Install Dependencies

```bash
npm install
```
#### 4. Start the Development Server
> If you want to run the project in a production environment, skip this part and move to the [Production Setup](#production-setup).
```bash
npm run dev
```
### Production Setup  
#### 1. Build the Project
> After completing steps 1–3 in the Getting Started section, run the following:

```bash
npm run build
```
#### 2. Start the Production Server

```bash
npm start
```
## Contributing

We welcome contributions to the Bralvio LLC website! Please follow the process below to ensure smooth collaboration and clean code.

### Contribution Workflow

#### 1. **Clone the repository**

   ```bash
   git clone https://github.com/Clarioscope-AI/bralvio.io.git
   cd bralvio.io
   ```
#### 2. Create a new branch

Use a descriptive name for your feature or fix:
```bash
git checkout -b feature/your-feature-name
```
#### 3. Make your changes

Implement your feature, fix bugs, or update documentation.
#### 4. Commit using Conventional Commits
Example:

```bash
git commit -m "feat: add contact form with validation
```
> Follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) to ensure consistency.

#### 5. Push your changes
#### 6. Create a Pull Request (PR)

- Open a PR against the main branch
- Assign the relevant reviewer (e.g., team lead)
- Add necessary labels/tags (e.g. feature, bug, enhancement)
- Ensure your PR title clearly describes the change
